* Overview
- *Goal:* turn one turntable MP4 of Peter's head into a 3D model (3D Gaussian Splatting, =.ply=).
- *Input:* subject rotates in place, camera fixed; background removed (or solid).
- *Output:* a standard 3DGS =.ply= (proportion-corrected) that opens directly in viewers like SuperSplat.
- *Key idea:* a turntable is a known circular orbit, so we *assign* camera poses analytically instead of
  *estimating* them (COLMAP). Then train plain 3DGS and apply a post-hoc proportion fix.
- *One line:* =mp4 → frames → masks → analytic poses (★rotation direction) → 3DGS → Z-rescale → .ply=

* Pipeline
#+begin_example
  turntable.mp4 ──[1] extract──▶ frames ──[2] trim still tail──▶ valid 1 revolution
        │                                                              │
        │                                              [3] foreground masks → RGBA
        ▼                                                              ▼
  [4] analytic turntable poses ★direction=-1, elev=0, r=3, fov=40 ─▶ transforms_*.json
        │                                                              │
        └──────────────[5] 3DGS training (free, no prior)─────────────┘
                                   ▼
                  point_cloud.ply  (vertically stretched: single-elevation limit)
                                   │ [6] post-hoc Z-rescale (match silhouette)
                                   ▼
                  *_zcorr.ply  ──[7] SuperSplat / turntable render
#+end_example

- *Why r3dg:* for a plain 3DGS one could use lighter, permissively-licensed tools (gsplat [Apache 2.0],
  the original INRIA gaussian-splatting). We chose r3dg because the program continues to *relight / edit /
  compose*: it is 3DGS-native and decomposes PBR material (albedo/roughness/normal) + environment light in
  the same pipeline — one consistent asset from geometry to relight.
- *License:* r3dg is research code (non-commercial, INRIA-rasterizer-based) → *PoC / internal demo only*.
  Production target: reimplement on gsplat (Apache 2.0).
- *GPU requirement — architecture (must) + VRAM:* prebuilt CUDA extensions target =sm_80/86=, so an
  *Ampere* GPU runs with no recompile.

* Steps
Each step is a single command and produces a concrete artifact.

** [1] Extract frames
#+begin_src bash
ffmpeg -i turntable_face.mp4 frames_raw/f_%04d.png
#+end_src
- *Output:* =frames_raw/*.png= (238 frames in our case).

** [2] Analyze rotation, trim the still tail
#+begin_src bash
python src/analyze_turntable.py frames_raw
#+end_src
- Detects frames where the turntable stopped (adjacent-frame correlation ≈ 1.0) and reports the valid
  single-revolution range; also saves a montage for a quick visual check.
- *Output:* valid frame range + =_contact.png= montage. *Always eyeball it* (front→back→front, uniform).

** [3] Foreground masks → RGBA
#+begin_src bash
python src/make_masks.py frames_raw rgba --bg auto --frames 0:214
#+end_src
- =--bg auto= picks black (luminance) or solid (color-distance) from the corners; cleanup keeps the
  largest connected component and fills holes.
- *Output:* =rgba/*.png= (masked images; training input).

** [4] Analytic turntable camera poses  ★critical★
#+begin_src bash
python src/poses_turntable.py data/peter_face/rgba data/peter_face --direction -1
#+end_src
#+ATTR_LATEX: :environment tabularx :width \linewidth :align llX :booktabs t
| Param         | Value | Meaning                                    |
|---------------+-------+--------------------------------------------|
| =direction=   | *-1*  | ★rotation sign; -1 for the Peter captures  |
| =elevation=   | 0     | eye-level (single elevation)               |
| =radius=      | 3.0   | camera distance (affects scale only)       |
| =fov=         | 40    | horizontal FOV (affects scale only)        |
| =test-every=  | 8     | every 8th frame held out for validation    |
- *Output:* =transforms_train.json= / =transforms_test.json= (per-frame camera pose + FOV; read by [5]).
- A wrong =direction= keeps only the front sharp and smears/concaves the rest (see Lessons).

** [5] 3DGS training (free, no prior)
#+begin_src bash
python -u train.py --eval -s data/peter_face -m outputs/peter_face/stage1 \
  --lambda_normal_render_depth 0.01 --lambda_normal_smooth 0.01 \
  --lambda_mask_entropy 0.1 --lambda_depth_var 1e-2 \
  --save_interval 30000 --checkpoint_interval 30000 --test_interval 7000
#+end_src
- Stock r3dg =train.py=, random init, no added prior. 30k iterations.
- *Output:* =point_cloud/iteration_30000/point_cloud.ply= + =chkpnt30000.pth= (for relight).

** [6] Post-hoc Z-rescale (proportion fix)
#+begin_src bash
python src/zrescale_ply.py <in.ply> outputs/peter_face/face_zcorr.ply 0.22
#+end_src
- The horizontal axes are correct; only Z is over-stretched. Measure true height:width from the silhouette,
  then squeeze Z by that factor (positions *and* each gaussian's covariance, re-decomposed).
- *Output:* =*_zcorr.ply= (final, proportion-correct 3DGS asset).

** [7] Inspect / render
#+begin_src bash
python src/render_turntable.py outputs/peter_face/face_zcorr.ply out_frames 300 0 800
ffmpeg -framerate 30 -i out_frames/f_%04d.png -c:v libx264 -pix_fmt yuv420p face.mp4
#+end_src
- Open =*_zcorr.ply= in SuperSplat; render a turntable video for sharing.
- *Output:* demo =*.mp4=.

* Configuration summary

| Step    | Key configuration                                                |
|---------+------------------------------------------------------------------|
| extract | ffmpeg, all frames                                               |
| trim    | adjacent-frame corr > 0.9999 = still                             |
| masks   | =--bg auto= (black>15 / solid>35) + fill + largest component     |
| poses   | *direction=-1*, elev=0, radius=3, fov=40, test-every=8           |
| train   | stock r3dg, 30k, random init, mask_entropy 0.1, depth_var 1e-2   |
| rescale | Z-rescale factor from silhouette (head ZF≈0.22), covariance-aware |
| render  | 300 frames / elev 0 / 800px → libx264 yuv420p                   |

* Lessons learned
- *Rotation direction is the single most important setting.* A reversed turntable direction keeps only the
  front sharp while everything else blurs and the surface caves inward; fixing it (no other change) makes
  the reconstruction sharp. If unsure, train both signs and keep the higher test-PSNR / lower sliver ratio.
- *Validate geometry, not PSNR.* A high PSNR can hide a badly distorted reconstruction; check silhouette
  proportions and gaussian shapes directly.
- *Don't fight the optimizer with strong priors.* Heavy geometric priors (hull containment, etc.) distorted
  the shape. The robust path was to fix the data/poses, train plainly, and correct proportions afterward.
- *Single-elevation capture cannot fix the vertical scale.* The result is vertically stretched by design;
  we correct it post-hoc with a silhouette-matched Z-rescale.
- *Face detail needs face pixels.* In a full-body frame the head is tiny, so face detail is poor — capture
  the face close-up (or higher resolution) when detail matters.

* Limitations & better capture
- Root limitation: a single-elevation turntable lacks vertical-scale information → requires the post-hoc fix.
- Better input: *multi-elevation capture* (several heights / helical path) → correct proportions with no
  post-processing; *uniform rotation* with no still frames; *1080p+* for detail. For a turntable the
  background carries no parallax (camera is fixed), so removing it is fine.
